## 2026.06.18

### 프로젝트 초기 세팅 문제

#### 증상
- Codex 실행 중 `gpt-5.3-codex` 모델이 ChatGPT 계정에서 지원되지 않는 오류가 발생했다.
- VS Code에서 사용하던 Codex 모델 설정이 현재 환경과 맞지 않았다.

#### 원인
- 현재 계정에서 지원되지 않는 모델을 선택했기 때문이다.
- VS Code 버전과 Codex 사용 모델을 현재 지원되는 조합으로 맞출 필요가 있었다.

#### 해결
- Codex 모델 설정을 지원되는 모델로 변경하기로 했다.
- VS Code를 최신 버전으로 업데이트했다.
- 이후 `5.4-mini` 버전으로 진행했다.

#### 학습 포인트
- Codex와 VS Code는 현재 지원되는 모델 조합을 맞춰야 정상 동작한다.
- 도구 버전이 오래되면 모델 선택이나 실행 과정에서 문제가 생길 수 있다.
- 초기 세팅 문제는 기능 문제가 아니라 환경 설정 문제일 수 있으므로 먼저 버전과 지원 여부를 확인해야 한다.

## 2026-06-20

### 데이터소스 분리 문제

#### 증상
- `application.yml`에 MariaDB 접속 정보가 들어 있어 GitHub에 노출될 수 있었다.
- 로컬 전용 설정을 분리했지만, 적용 방식이 처음에는 헷갈렸다.

#### 원인
- 공통 설정과 로컬 비밀값을 같은 파일에 두었기 때문이다.
- `application-local.yml`은 `local` 프로필이 활성화되어야 적용된다.

#### 해결
- DB 설정을 `application-local.yml`로 분리했다.
- `.gitignore`에 `application-local.yml`을 추가했다.
- `application.yml`에 `spring.profiles.active: local`을 넣어 로컬 설정이 기본 적용되게 했다.

#### 학습 포인트
- 공통 설정과 비밀 설정은 분리하는 것이 안전하다.
- Spring Boot는 프로필 기반으로 설정 파일을 합쳐서 읽는다.
- 실행 명령이 아니라 설정 파일 중심으로 기본 동작을 정해둘 수 있다.
